About

This workshop brings together researchers from Peking University and Japanese universities to share recent advances in bidirectional transformations (BX), lens theory, database languages, and LLM-assisted software engineering. The event is organized as part of the ongoing joint research project on trustworthy and ubiquitous information systems based on bidirectional computing.

Talks cover a broad spectrum — from foundational theory of well-behaved lenses and derivative-based BX, to practical applications in SQL view update, collaborative editing, program optimization with LLMs, and graph-augmented software development.

Zhenjiang Hu PKU
Derivative-based Bidirectional Programming
Approaches to developing BXs can be mainly categorized into get-based and put-based approaches, which suffer from ambiguity and specification complexity, respectively. In this paper, we propose a novel derivative-based framework to bidirectional programming, where BX is specified through get′, the derivative of get. From get′, we systematically derive well-behaved BXs via integration and right inversion, and prove completeness for all well-behaved BXs. We demonstrate its usefulness in bidirectional programming on relational databases and trees. (Joint work with Guanchen Guo, Xiao He, and Tao Zan.)

Haiyan Zhao PKU
Bridging the Semantic Gap in Software Development with Graph-Augmented LLMs
Large language models (LLMs) have demonstrated remarkable capabilities in general software engineering tasks, yet they face fundamental challenges when applied to repository-level scenarios: (1) the semantic gap between natural language issue descriptions and source code, which manifests as symptom-to-cause and one-to-many mismatches in issue localization; and (2) the lack of repository-specific knowledge (e.g., local APIs, coding styles, dependency patterns) in pre-trained LLMs for code generation/completion. To address the challenges, we propose a fine-grained, graph-based modeling approach for code repositories, presenting recent work on core repository-level tasks that correspond to different phases of the repository lifecycle through Graph-Augmented LLMs.

Yingfei Xiong PKU
A DSL for Program Optimization
With the development of large language models (LLMs), more and more program optimization tasks could be delegated to LLMs. However, purely relying on LLMs for optimizing large software projects has two drawbacks: (1) Correctness — LLM optimization does not ensure correctness, and optimized code still needs to be checked by programmers, creating a bottleneck; (2) Cost — scanning and optimizing a whole software project with an LLM is expensive, preventing frequent use. Therefore, instead of asking an LLM to directly optimize the program, we ask an LLM to write optimization rules, and design a domain-specific language (DSL) for such rules. The DSL features verifiability and LLM-affinity: each rule can be automatically verified, and errors reported by the verifier are interpretable at the surface level for the LLM.

Fusen Wang PKU
View-based Collaborative Editing via Lenses
Collaborative editing systems, exemplified by Google Docs, have revolutionized collaboration by enabling simultaneous updates and strong eventual consistency with offline support. However, these systems often lack fine-grained access control, requiring all participants to maintain full data replicas. This leads to two critical issues: privacy risks from exposing sensitive data to unauthorized collaborators, and excessive cognitive load for users who must manually filter irrelevant information. We propose a new idea of view-based collaborative editing, and present a general framework integrating CRDTs and Bidirectional Transformations (lenses). We introduce a lens source-lifting algorithm that extends existing lenses to support replicated data types on the source side, and design a concurrent synchronization protocol to maintain consistency between sources and views across distributed nodes. We have implemented our approach in ViewSync, a prototype for view-based collaborative JSON editing.

Makoto Onizuka Japan
1) TraceableRAG: A Framework for Traceable and Evidence-Grounded Generation, 2) Hard to Read, Hard to Maintain? Refactoring Complex SQL into Wvlet Pipelines
1) TraceableRAG treats LLM dialogue histories as a context database for future generation tasks. It addresses three requirements: selecting contexts relevant to each artifact section, generating statements grounded only in retrieved evidence, and tracing user feedback back to the responsible contexts and generation rules. The framework segments and labels dialogue histories, performs section-aware retrieval, applies policy- and evidence-constrained generation, and records block-level provenance. When users identify an error, TraceableRAG diagnoses its cause, updates the retrieval knowledge or generation policy, and regenerates only the affected content. 2) Modern analytical SQL queries are often hard to read and maintain because of duplicated logic, deep nesting, and the gap between clause order and logical data flow. We present Wvlet Cognitive Workbench (WCW), a browser-based system for readability-oriented query refactoring. WCW transforms SQL into Wvlet, improves query structure through duplicate extraction and safe join flattening, and supports side-by-side comparison with readability metrics and semantic equivalence checking. Our system also supports workload-level refactoring by extracting shared reusable models across multiple queries.
